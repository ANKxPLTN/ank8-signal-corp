Core Components

1. Learning Engine

The Learning Engine is the brain of ANK8. It manages curriculum delivery, tracks user progress, and generates personalized recommendations.

Responsibilities: Store and deliver learning modules, track completion, generate learning paths.

Data Flow: User Action → Agent Observation → Learning Engine Processing → Recommendation Generation → GUI Display → User Confirmation

2. AI Coordination System

The AI Coordination System processes queries, provides explanations, and generates guidance.

Security Model: Read-only analysis only, No autonomous command execution, Explicit user confirmation required, All suggestions logged and auditable.

3. Server Agent

Deployed directly on user's VPS, the Server Agent is the "eyes and ears" of ANK8.

Installation Method:

curl -sSL https://install.ankverdict.io/agent | bash

Agent Limitations (Intentional):

  • [ALLOWED] Can read system files
  • [ALLOWED] Can observe commands
  • [ALLOWED] Can analyze configurations
  • [DENIED] Cannot modify files
  • [DENIED] Cannot execute commands
  • [DENIED] Cannot install packages
  • [DENIED] Cannot modify system settings

4. Data Storage Layer

User Data: Account info, Server configurations, Learning progress, Custom goals.

Infrastructure Data: Hardware specs, OS versions, Network status.

Encrypted at Rest: AES-256 encryption, Key management via AWS KMS.

5. Audit & Compliance Logging

Every action is logged for accountability (Authentication, API calls, Commands executed).

Retention: 90 days (Hot storage), 1 year (Archive), 7 years (Legal hold).

Communication Protocol

User ↔ Platform Communication

Protocol: REST API with WebSocket fallback.

Authentication: API Key + User JWT Token.

Encryption: TLS 1.3 with perfect forward secrecy.

Agent ↔ Platform Communication

Frequency: Health check (5 min), Metrics push (10 min), Event reporting (Real-time).

Payload Compression: GZIP compression, Batch operations, Delta sync.

Security Architecture

Defense-in-Depth Strategy

  • Layer 1: Network Security: VPC isolation, DDoS protection (AWS Shield).
  • Layer 2: Application Security: Input validation, SQL injection prevention, XSS protection.
  • Layer 3: Data Security: Encryption at rest (AES-256) and transit (TLS 1.3).
  • Layer 4: Agent Security: Code signing verification, Sandboxed execution, Permission model.

Threat Model

Threats Considered: Man-in-the-Middle Attacks, Data Breach, Malicious Agent, Unauthorized Access, Privilege Escalation, Data Exfiltration.

Scalability Design

Horizontal Scaling

Learning Engine (Stateless), AI System (Load-balanced), API Gateway (Kubernetes), Database (Sharded).

Target Capacity: 10,000 concurrent users, 50,000 servers managed, 1M queries per day.

Deployment Architecture

Cloud Infrastructure

Provider: AWS (primary), with multi-region redundancy.

Services: ECS/Kubernetes (Compute), RDS PostgreSQL (Database), S3 (Storage), CloudFront (Networking).

Disaster Recovery

RTO: 15 minutes. RPO: 5 minutes. Multi-region deployment, Active-active API, Automated failover.
